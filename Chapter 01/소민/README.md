# 1. 리액티브 시스템과 리액티브 프로그래밍

리액티브 시스템: 사용자의 요청에 즉각적으로 반응을 한다.(여기에다가 확장성과 유지보수도 용이해야함) → **비동기 메시지 통신을 기반으로 함**

- 비동기 메시지 기반으로 동작
- 회복성이 높아야함
- 높은 응답성(빠른 응답)
- 유지보수와 확장 용이

리액티브 프로그래밍: 리액티브 시스템을 구축하는데 필요한 프로그래밍 모델 → 비동기 메시지 통신이므로 non-blocking I/O 를 기본으로 한다.

- data streams: 데이터 흐름, 즉 데이터가 지속적으로 발생한다는 것
- the propagation of change: 데이터가 발생할 때 마다 이걸 변화하는 이벤트로 취급 → 이벤트를 발생시키면서 데이터를 계속적으로 전달한다.
- 선언형 프로래밍: 동작의 수행 방식을 구체적으로 명시하지 않고, 액션만 명시한다. (e.g., list 에 담긴 수를 더할 때 for 문을 돌면서 더하느 코드는 명령형, java stream 을 사용해서 sum() 을 하는건 목표(더하기)만 명시하므로 선언형)
    - 선언형에서는 메서드 체이닝을 통해 한 문장으로 코드를 구성할 수 있다.

### 리액티브 핵심 구성요소

> publisher

입력으로 들어오는 데이터를 제공한다.

> subscriber

publisher 가 주는 데이터를 받아서 사용한다.

> data source == data stream

데이터가 연속적으로 끊임없이 들어오는 것, 입력으로 들어오는 데이터

⇒ publisher 가 입력으로 받는 데이터

> operator

publisher 와 subscriber 사이에서 가공 처리를 하는 동작(가공을 담당하는 역할이 operator)
